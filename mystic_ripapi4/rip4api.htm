3.1 Constructor / Destructor


Constructor Create

Allocates the 640x350 pixel buffer (224,000 bytes on heap).

Initializes all 10 CHR font slots to Nil.

Initializes all 10 saved screen slots to Nil.

Sets clipboard to Nil.

Calls Reset to set default state.

After Create, the engine is ready to accept commands.


Destructor Destroy

Frees all loaded CHR fonts (Dispose on non-Nil slots).

Frees all saved screens (Dispose on non-Nil slots).

Frees clipboard (FreeMem if non-Nil).

Frees the pixel buffer.

Call Free (not Destroy directly) per standard Pascal convention.


3.2 RIP Command Processing


Procedure ProcessLine (Line: String)

Feed a line of RIP data. Handles backslash (\) line continuation.

Extracts all !| (and #1|, #2|) command sequences and dispatches

each one through ProcessCommand. Non-RIP text is ignored.


Procedure ProcessCommand (Cmd: String)

Process a single RIP command (the text after !|). Parses level

digit(s) and command character, dispatches to Level 0, 1, or 9.


3.3 Screen Management


Procedure Reset

Restores all state to defaults:

Position: (0,0)

Draw color: 15 (white), fill color: 0 (black)

Fill style: solid, line style: solid, line thickness: 1

Write mode: copy (overwrite)

Font: default 8x8, size 1, horizontal, left/top justify

Text window: (0,0)-(79,42), size 0

Viewport: (0,0)-(639,349)

Palette: default EGA 16-color

Fill pattern: all $FF (solid)

Button style: BevelSize=1, DFore=15, BRight=15, DDark=8,

Surface=7, CornerCol=16 (disabled)

Mouse fields: all cleared (count=0, all records zeroed)

Text variables: all cleared

CHR fonts: all freed and set to Nil

Saved screens: all 10 slots freed

Saved text window / mouse fields: cleared

Clipboard: freed

Line buffer: cleared

Clears the screen after resetting state.


ParseLevel0 '*' (RIP_RESET_WINDOWS) resets text window,

viewport, TextWinSize, and kills all mouse fields. This is

a partial reset — it does NOT reset colors, fonts, or styles.


Procedure ClearScreen

Zeros the entire 640x350 pixel buffer (all pixels to color 0).


Procedure ClearViewport

Zeros only the pixels within the current viewport rectangle.

3.5 Text Output


Procedure OutTextXY (X, Y: SmallInt; S: String)

Draw text at (X,Y). Expands $VARNAME$ text variables via ExpandVars

before rendering. If a CHR font is loaded for the current font

number, uses vector stroke rendering. Otherwise uses 8x8 CP437

bitmap font. RIP command: !|@


The engine's ParseLevel0 handler for '@' calls OutTextXY directly,

ensuring all text rendered by RIP commands gets variable expansion

and CHR font support.


Procedure OutText (S: String)

Draw text at current position. Same variable expansion and font

selection as OutTextXY. RIP command: !|T


ParseLevel0 'T' and ParseLevel1 't' (region text) both route

through OutText/OutTextXY. This means text variables like $CURX$,

$DATE$, user-defined variables, etc. are expanded automatically

in all text rendering paths.


Procedure DrawTextCHR (X, Y: SmallInt; S: String; AFont, ASize: Byte)

Render text using a specific loaded CHR vector font at the given

scale. AFont = font slot (1-10), ASize = magnification (1+).

Each character is rendered as a series of move/draw stroke commands.

Does NOT expand variables — called after expansion by OutText.

3.13 Image Operations


Procedure GetImage (X0, Y0, X1, Y1: SmallInt; Var Buf)

Copy screen region to buffer. Buffer format: width(Word) +

height(Word) + pixel data (one byte per pixel, row-major).

Caller must allocate buffer with ImageSize bytes.

RIP command: !|1C


ParseLevel1 'C' handler allocates a heap clipboard buffer via

GetMem and calls GetImage to capture the region. Any previous

clipboard is freed first.


Procedure PutImage (X, Y: SmallInt; Var Buf; Mode: Byte)

Draw captured image at (X,Y). Reads width/height from buffer

header, then renders pixel data. Mode = write mode (0=copy, 1=XOR).

RIP command: !|1P


ParseLevel1 'P' handler pastes the heap clipboard to screen

if one exists.


Function ImageSize (X0, Y0, X1, Y1: SmallInt) : LongInt

Calculate buffer size needed for GetImage: (W * H) + 4 bytes

(4 bytes for width + height header).


3.14 Icon Files (BGI GetImage Format)


All icon files (ICN, MSK, HIC) use the Borland BGI GetImage format:


Header: width-1 (Word) + height-1 (Word)

Data: 4 EGA bit planes interleaved per scanline

Plane order: 3, 2, 1, 0 (MSB first)

Each plane: ceil(width/8) bytes per row

Footer: 2 bytes padding (ignored)


Example: a 22x17 icon has header 21,16 and data = ceil(22/8) * 4 * 17

= 3 * 4 * 17 = 204 bytes + 2 padding = 206 total.


Function LoadIcon (FileName: String; X, Y: SmallInt; Mode: Byte) : Boolean

Load .ICN file and render at (X,Y). Mode = write mode (0=copy, 1=XOR).

Reads BGI header, decodes 4 EGA planes per row, combines plane bits

into 4-bit color index per pixel, calls DrawPixel for each.

Returns False if file not found or header invalid.

RIP command: !|1I


Function SaveIcon (FileName: String; X0, Y0, X1, Y1: SmallInt) : Boolean

Save screen region as .ICN file. Writes BGI header (w-1, h-1),

splits each pixel into 4 EGA planes, writes interleaved, adds

2 bytes padding. RIP command: !|1W


3.15 Icon Masks and Highlights (Phase 2)


Function LoadMask (FileName: String; X, Y: SmallInt) : Boolean

Load a .MSK mask file and apply as AND transparency mask.

Same BGI GetImage format as ICN. Where all 4 planes are zero

(mask value = 0), the corresponding screen pixel is cleared to

color 0 (transparent). Where any plane bit is set (mask > 0),

the pixel is left unchanged (opaque).


Usage: draw icon first, then apply mask to punch out transparent

areas. The mask file must be the same dimensions as the icon.


MSK files in RIPterm154: BUBBLE.MSK, CHROME.MSK, MOUSE.MSK, TANK.MSK


Function LoadIconMasked (IconFile, MaskFile: String; X, Y: SmallInt) : Boolean

Convenience: loads icon at (X,Y) then applies mask on top.

Internally calls LoadIcon followed by LoadMask.

Returns False if icon file fails to load.


Function LoadHotIcon (FileName: String; X, Y: SmallInt) : Boolean

Load a .HIC highlighted icon file. Same BGI GetImage format.

Used for button hover/selected states — the HIC replaces the

normal ICN when the button is selected or the mouse hovers.


HIC files in RIPterm154: BUTTON.HIC, RADIO.HIC, RADIOBUT.HIC

Full-featured button creation with icon and state support.

If empty, label text is drawn via OutTextXY

Procedure ClickButton (Index: Integer)

Simulate a click on button at mouse field Index.


Radio button: Deselects all other buttons in the same GroupID.

Selects this button. Swaps to hot icon if available,

otherwise XOR inverts the region.


Checkbox: Toggles Selected state. Swaps between normal icon

and hot icon, or XOR inverts/un-inverts.


Plain button: XOR inverts the region (visual click feedback).


Does nothing if Index is out of range, inactive, or not a button.

bits, effectively flipping each color to its complement

(0<->15, 1<->14, etc). Used for button click feedback and

the INVERT flag in mouse regions.

Load Borland BGI .CHR file into font slot AFontNum (1-10).

3.19 System Font Modes (Phase 6)


RIPterm v1.54 supports 5 system text modes, selected via the

TextWinSize parameter of SetTextWindow (RIP command !|w).


Mode Cols Rows Font Pixels

0 80 43 8x8 Default EGA mode

1 80 25 8x14 Standard VGA text

2 40 25 16x14 Double-width (each pixel drawn twice)

3 91 43 7x8 Narrow columns

4 91 25 7x14 Narrow tall


Font data files:

rip_font8x8.inc — 2048 bytes, 256 chars, CP437 8x8 VGA ROM

rip_font8x14.inc — 3584 bytes, 256 chars, CP437 8x14

(derived from 8x8: 3 blank rows top, 8 glyph rows, 3 bottom)


DrawText8x8 selects the font based on TextWinSize:

- Modes 0, 3: use Font8x8 data, CharW pixels wide

- Modes 1, 2, 4: use Font8x14 data, CharH=14

- Mode 2: double-width, each font pixel drawn as 2 screen pixels

- Modes 3, 4: 7 of 8 font columns rendered (narrow)


Function GetSysFontW : Integer

Character width in pixels for current mode:

Mode 0: 8, Mode 1: 8, Mode 2: 16, Mode 3: 7, Mode 4: 7


Function GetSysFontH : Integer

Character height in pixels:

Modes 0, 3: 8. Modes 1, 2, 4: 14.

3.21 Image Loading (Phase 4)


Function LoadPCX (FileName: String; X, Y: SmallInt) : Boolean

Load a 16-color EGA PCX file and render at (X,Y).


Supported format:

Manufacturer: 0x0A (ZSoft)

Encoding: RLE (type 1)

Bits per pixel: 1

Planes: 4 (= 16 colors)

Max dimensions: 640x350


Decodes RLE-compressed scanlines: if top 2 bits of a byte are set,

lower 6 bits = run count, next byte = repeated value. Otherwise

the byte is a literal pixel value.


Each scanline has 4 planes of BytesPerRow bytes. Planes are combined

into 4-bit color indices the same way as ICN/MSK files.


Tested with RIPTERM.PCX (640x301 EGA splash screen).


Function LoadBMP (FileName: String; X, Y: SmallInt) : Boolean

Load a 4-bit (16-color) or 24-bit BMP file and render at (X,Y).


4-bit mode: reads nibble-packed pixels directly as EGA color indices.

Handles row padding to 4-byte boundaries. Supports bottom-up and

top-down row order.


24-bit mode: reads BGR pixel data and maps each pixel to the nearest

EGA color using Manhattan distance against the EGA_RGB palette table.

This allows importing screenshots or external images into the

640x350 EGA canvas.


Returns False if file format is unsupported or dimensions exceed

640x350.


3.22 Screen State Save/Restore


The RIPterm save/restore system allows complete graphics state to be

preserved and recalled. BBS software uses this to save the screen

before displaying a sub-menu or dialog, then restore it when done.


Procedure SaveScreen (Slot: Byte)

Save the entire 640x350 pixel buffer to a memory slot (0-9).

Allocates 224,000 bytes on the heap per slot. If the slot already

contains a saved screen, it is overwritten.

Text variable: $SAVE$ (slot 0), $SAVE0$ through $SAVE9$


Procedure RestoreScreen (Slot: Byte)

Restore a previously saved screen. The saved data is copied back

to the pixel buffer and the slot is freed. Per RIPterm behavior,

numbered slots (0-9) are deleted after restore. If the slot is

empty, the call is silently ignored.

Text variable: $RESTORE$ (slot 0), $RESTORE0$ through $RESTORE9$


Procedure SaveTextWin

Save the current text window coordinates (X0, Y0, X1, Y1) and

font size. Only one text window state can be saved at a time;

a second call overwrites the first.

Text variable: $STW$


Procedure RestoreTextWin

Restore the previously saved text window state. The saved data

is cleared after restore. If nothing was saved, call is ignored.

Text variable: $RTW$


Procedure SaveMouseAll

Save all active mouse field definitions (up to 128 fields),

the current mouse count, NextTabIndex, and FocusedField.

All Phase 2/5 fields (HotKey, TabIndex, IsRadio, etc) are

preserved via full record copy. Only one mouse state at a time.

Text variable: $SMF$


Procedure RestoreMouseAll

Restore all previously saved mouse fields. Calls UnfocusField

first to remove any current highlight, then restores MouseCount,

NextTabIndex, FocusedField, and all field records. Current fields

are replaced entirely. Saved data is cleared after restore.

Text variable: $RMF$


Procedure SaveClip

Save the current GetImage clipboard to a backup buffer.

If a backup already exists, it is freed first. Saves the

clipboard data, size, and dimensions (ClipW/ClipH).

Text variable: $SCB$


Procedure RestoreClip

Restore the saved clipboard. Frees current clipboard and

replaces it with the backup copy. Clipboard data, size,

and dimensions are all restored.

Text variable: $RCB$


Procedure SaveAll

Shorthand for: SaveScreen(0) + SaveTextWin + SaveClip + SaveMouseAll.

Text variable: $SAVEALL$


Procedure RestoreAll

Shorthand for: RestoreScreen(0) + RestoreTextWin + RestoreClip +

RestoreMouseAll.

Text variable: $RESTOREALL$


Memory Management:

Screen saves are heap-allocated on demand. Each slot = 224KB.

Full set of 10 slots = 2.2MB. Freed on restore or engine Destroy.

Clipboard uses GetMem/FreeMem. All memory cleaned up in Destroy.


Text Variable Quick Reference:

$SAVE$ Save screen to slot 0 (persistent)

$RESTORE$ Restore screen from slot 0 (persistent)

$SAVE0$..$SAVE9$ Save screen to numbered slot

$RESTORE0$..$RESTORE9$ Restore and delete numbered slot

$STW$ Save text window state

$RTW$ Restore text window state

$SCB$ Save clipboard

$RCB$ Restore clipboard

$SMF$ Save mouse field definitions

$RMF$ Restore mouse field definitions

$SAVEALL$ Save all (screen + textwin + clip + mouse)

$RESTOREALL$ Restore all

$RESET$ Full engine reset

Scope Levels:
  LOCAL (0)    — cleared on scene end (ClearScreen)
                 Used for: temp values within a single screen
  SESSION (1)  — persists for connection duration
                 Used for: user prefs, navigation, form data

                 Used for: saved settings, statistics

  Connection starts
    Load persistent vars from disk
      Scene loads
        Local vars created
        Scene ends (ClearScreen) → local vars cleared
      Next scene loads
        Session vars still available
      User disconnects
        Save persistent vars to disk
        Session vars cleared
