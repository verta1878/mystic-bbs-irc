
,----------------------.
| UPGRADING MYSTIC BBS |------------------------------------------------------
`----------------------'

 1. FIRST and ALWAYS create a backup of your BBS before upgrading.  Generally
    upgrading goes pretty smooth, but there is always the chance of something
    going wrong whether it is Mystic or operator error!

 2. There is no automatic upgrade process.  The process is to install the full
    release and then copy over your existing data files and themes.

 MYSTIC BBS 1.10IRC UPGRADE NOTES
 ==================================

 If upgrading from stock Mystic 1.10 A38 or earlier:

 1. Create a "webroot" directory under your Mystic root:

      Windows:  C:\mystic\webroot\
      Linux:    /mystic/webroot/

    The HTTP file server (port 8080) serves static web pages from this
    directory. Place an index.htm file here for your BBS web page.
    If the directory does not exist, HTTP requests to / will show
    a default page with your BBS name.

 2. Protocol configuration (optional):

    If you want internal file transfer protocols instead of external,
    add these entries in the Protocol Editor:

      Key=X  Desc=Xmodem       Batch=No   SendCmd=@XMODEM   RecvCmd=@XMODEM
      Key=Y  Desc=Ymodem       Batch=Yes  SendCmd=@YMODEM   RecvCmd=@YMODEM
      Key=G  Desc=Ymodem-G     Batch=Yes  SendCmd=@YMODEMG  RecvCmd=@YMODEMG
      Key=Z  Desc=Zmodem       Batch=Yes  SendCmd=@ZMODEM   RecvCmd=@ZMODEM
      Key=8  Desc=Zmodem 8K    Batch=Yes  SendCmd=@ZMODEM8  RecvCmd=@ZMODEM8
      Key=3  Desc=Zmodem 32K   Batch=Yes  SendCmd=@ZMODEM32 RecvCmd=@ZMODEM32
      Key=K  Desc=Kermit        Batch=Yes  SendCmd=@KERMIT   RecvCmd=@KERMIT

    External protocols (DSZ, etc) still work as before.

 3. MIS now includes an HTTP file server on port 8080.

    It starts automatically with MIS. It serves:
      - Static pages from webroot/ (index.htm, CSS, images)
      - File downloads via FTP Name mapping (/FtpName/filename)

    FTP Name and FTP ACS must be configured per file base in the
    File Base Editor for files to be accessible via HTTP or FTP.

 4. FTP server fixes applied:

    - SIZE command now returns actual file size (was "not implemented")
    - REST command added for resume support
    - PASV endian fix for passive mode connections

 5. default.txt (language/prompts):

    This release uses prompts 0-515. If you have a custom theme,
    verify your prompt file has at least 516 entries (prompts 0-515).
    Prompts above 515 from newer Mystic versions are not used yet.

 6. MPL scripts may need recompilation if upgrading from an older version.
    Run your .mps files through the MPL compiler to regenerate .mpx files.
