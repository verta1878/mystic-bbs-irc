RIPscrip Protocol Versions — Road Map
===============================================================================

RIPscrip v1.54 (IMPLEMENTED — this engine):

  * MegaNum encoding (base-36, coordinates 0-1295)
  * 640x350 EGA 16-color rendering
  * BGI-compatible drawing primitives
  * Borland CHR vector fonts (10 slots)
  * 5 system bitmap font modes (8x8, 8x14, 16x14, 7x8, 7x14)
  * BGI GetImage format icons (ICN, MSK, HIC)
  * PCX and BMP image loading
  * 3D beveled button system with radio/checkbox/hotkey/tab
  * Screen save/restore (10 slots + text window + mouse + clipboard)
  * Text variable system with $VARNAME$ expansion (43 built-in)
  * Variable persistence (SaveVars/LoadVars)
  * Scene serialization (SaveScene/LoadScene)
  * Mouse field tracking (128 fields)
  * Line continuation, multi-command lines
  * Published by TeleGrafix Communications, 1993-1994

RIPscrip 2.0 (achievable):

  * UltraNum encoding (base-64, larger coordinates up to 4095)
  * Higher resolutions beyond 640x350
  * Mostly backward-compatible with v1.54

RIPscrip 3.0 (vaporware — partial at best):

  * Resolution-independent world coordinates
  * True color (24-bit)
  * JPEG/GIF/PNG image support
  * Scalable fonts
  * Multimedia (sound, animation)
  * Data tables
  * Never fully standardized or widely implemented

===============================================================================
RIPscrip 2.0 Feature Analysis (from RIPaint 2.1 evaluation archive)
===============================================================================

No code was taken. All information derived from file format observation,
FILE_ID.DIZ description, and THANKYOU.DOC documentation text.

NEW FILE FORMATS:

  .BMH  — BMP Highlight icons. Standard BMP files (not BGI GetImage).
           Same dimensions as v1.54 HIC files but stored as Windows BMP
           with 4bpp (16-color). Sizes match: BUTTON.BMH = 22x17 4bpp,
           RADIO.BMH = 15x11 4bpp, RADIOBUT.BMH = 25x17 4bpp.
           Replaces .HIC for button hot-state icons.

  .RFF  — RIPscrip Font Format. Scalable vector font with:
           - Design units (1000 unit em square)
           - Multiple faces/styles per font file (up to 380 records)
           - Style variants: Regular, Thin, Condensed, Wide, Extended,
             Horizontal, plus hot-state versions (HT, HC, HW, HE)
           - 224 characters (codes 32-255)
           - Font names: Cobb, DEFAULT, Dixon, Marin, Symbol
           - Much larger than CHR (31-62KB vs 5-18KB)
           - Format version 2.2

  .PAL  — Palette files. Two formats:
           16-byte: EGA palette indices (v1.54 compatible)
           868-byte: 256-color RGB palette with 100-byte header
             Header: "RIPaint 2.0 Palette File" signature
             Data: 256 x RGB triplets (768 bytes)

  .JPG  — JPEG image support (new in 2.0)
           Sample files: ASTRO.JPG, EAGLE.jpg, JUPITER.JPG, etc.

  .WAV  — WAV audio playback
           Sample files: RWSOUND1.WAV, RWSOUND2.WAV, RWSOUND3.WAV
           HMI audio drivers (HMIDET.386, HMIDRV.386)

  .PAT  — Fill pattern presets (8 bytes each, same as v1.54 format)
           55+ patterns included (BRICKS, HONEYCMB, LATTICE, etc.)

  .BUT  — Button style presets (128 bytes)
  .DSH  — Dash/line style presets (128 bytes)
  .GRD  — Grid presets (128 bytes)

EXTENDED COLOR:

  256-color palette support (up from 16)
  Color values in RIP commands use base-36 encoding beyond 0F:
    0G, 0H, 0I... 1V, 2R, 3N... up to color 255
  RAINBOW.PAL provides full 256-color RGB mapping

EXTENDED RESOLUTION:

  640x350  EGA (v1.54 compatible)
  640x480  VGA
  800x600  SVGA
  1024x768 SVGA
  Resolution-independent coordinate system via design units

NEW RIP COMMANDS (observed in RIP2EXAM.RIP):

  |J  — Jump/goto (flow control?)
  |n  — Set resolution/mode?
  |M  — Extended move? (different param count from v1.54)
  |f  — Font select (RFF fonts?)
  |k  — Extended color set?
  |N  — New drawing context?
  |K  — Extended kill/reset?

ANIMATION:

  Frame-based BMP animation (EARTH_01..EARTH_30.BMP = 30-frame globe)
  RWANIM01..RWANIM06.BMP = 6-frame animation sequence
  FADEIN.RIP / FADEOUT.RIP = transition effects (69KB each!)

ACHIEVABLE ADDITIONS TO OUR ENGINE:

  1. LoadBMH — trivial, BMH files are standard 4bpp BMP (our LoadBMP
     already handles this, just need to accept .BMH extension)

  2. 256-color palette — extend Palette[0..15] to Palette[0..255],
     pixel buffer stays Byte-per-pixel (already is)

  3. Extended MegaNum — support 3+ digit base-36 for coords > 1295
     (our MegaNum already handles variable digit counts)

  4. Higher resolutions — make canvas width/height configurable
     (currently hardcoded 640x350)

  5. PAL file loading — 16-byte or 868-byte format

  6. JPEG loading — would need a JPEG decoder library

  7. RFF font loading — complex but documented structure above

SVGACC Library Reference (with author permission)
===============================================================================

Zypher Software SVGACC v2.6 — 122-function SVGA graphics library.
Copyright (c) 1993-2000 Stephen L. Balkum and Daniel A. Sill.
Source code rights granted by author for backporting to Pascal.

Functions relevant to RIPscrip 2.0 engine (by category):

  Block Operations:
    blkget(x1,y1,x2,y2,blk)     — capture screen region
    blkput(mode,x,y,blk)         — paste with pixel mode
    blkresize(w,h,src,dst)       — resize raster block
    blkrotate(ang,fill,src,dst)  — rotate raster block

  2D Transformations:
    d2rotate(pts,xo,yo,ang,in,out)   — rotate point array
    d2scale(pts,xs,ys,in,out)        — scale point array
    d2translate(pts,xt,yt,in,out)    — translate point array

  3D Transformations:
    d3rotate(pts,xo,yo,zo,zr,yr,xr,in,out) — 3D rotation
    d3scale(pts,xs,ys,zs,in,out)            — 3D scale
    d3translate(pts,xt,yt,zt,in,out)        — 3D translate
    d3project(pts,params,in,out)            — 3D projection

  Drawing:
    drwline, drwaline (aliased)    — line with pixel mode
    drwbox, drwfillbox             — rectangle outline/filled
    drwcircle, drwfillcircle       — circle outline/filled
    drwellipse, drwfillellipse     — ellipse outline/filled
    drwcirarc, drwellarc           — circular/elliptical arc
    drwcubicbezier                 — cubic bezier curve
    drwpoint                       — single pixel
    drwstring (4 directions)       — text rendering
    fillarea, fillcolor            — flood fill by border/color
    fillconvexpoly, fillpoly       — polygon fill

  Palette:
    palget, palset                 — get/set palette (256 RGB)
    palchgauto, palchgstep         — palette fade/transition
    paldimstep                     — dim palette by percentage
    palrotate                      — rotate palette entries
    palcopy                        — copy palette range

  Font:
    fontset(font)                  — set custom bitmap font
    fontsystem                     — restore system font
    fontgetinfo(w,h)               — get current font size

  Image:
    pcxgetinfo, pcxput, pcxmake    — PCX load/save

  Sprite:
    spriteget, spriteput           — sprite with transparency
    spritegap                      — sprite with background save
    spritecolldetect               — collision detection

  Scroll:
    scrollup, scrolldn, scrolllt, scrollrt — region scrolling

  Viewport:
    setview(x1,y1,x2,y2)          — clipping region

  Data Structures:
    RasterBlock { width, height, data[] }  — pixel buffer
    D2Point { x, y }                       — 2D coordinate
    D3Point { x, y, z }                    — 3D coordinate
    ProjParameters { eyex-z, scrd, theta, phi } — 3D projection
    RGB { r, g, b }                        — palette entry
    PaletteData[256]                       — full 256-color palette
    Font { width, height, data[4096] }     — bitmap font
    PixelMode { SET, XOR, OR, AND }        — pixel operations
