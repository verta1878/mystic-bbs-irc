# mystic_misdos — a DOS-style MIS "Waiting For Caller" example

An **optional, self-contained example** that recreates the classic Mystic
1.06 **Waiting-For-Caller (WFC)** screen and drives it with the fork's two
communication add-ons: the modem code (`mystic_modem`) and the BinkP/mailer
code (`mystic_mailer`).  Every option shown on the WFC screen is functional.

It is **deliberately separate from the shipping MIS server** in `mystic/`
(`mis.pas`).  That MIS is the telnet daemon; this is the *DOS-MIS* teaching
example — a local console front-end that waits for a modem/BinkP caller or a
local login, exactly like Mystic's old DOS WFC.  (Sysop decision,
2026-07-08: keep MIS-DOS out of the main source.)

## The screen

`wfc.ans` is an 80x25 CP437 ANSI reconstruction of the 1.06 WFC: a title
bar, a **Node Listing** panel, a **Modem Info** panel, a **Local Mode** tab,
and a **System Commands** grid with a live status column (Node / OS /
Overlay / Next Event) and a clock + date.  It is regenerated by
`../mystic_modem/gen_wfc.py` (the same art the modem example uses), so the
two examples share one canonical WFC.

## What each key does

| Key | Action |
|-----|--------|
| U S P E # M G A V F L | open the matching Mystic editor (runs the config tool if present; otherwise says which editor it is) |
| X | **Answer Modem** — takes the line off-hook via `mystic_modem` |
| D | **Drop to DOS** — spawns a subshell, returns to the WFC on exit |
| Q | **Quit to DOS** — leaves the WFC loop |
| SPACE | **Local Login** — runs a local node session |

On a real modem CONNECT the loop sniffs the line: a **BinkP** caller is
handed to the `mystic_mailer` BinkP seam (mail transfer), a human caller
gets a session hook, and EMSI/others are reported.

## Files

- `misdos.pas`          — the program: draws the WFC, ticks the clock, watches
                          the modem, and dispatches keys.
- `misdos_screen.pas`   — loads `wfc.ans` and fills the live fields.
- `misdos_commands.pas` — the hot-key handlers.
- `wfc.ans`             — the WFC art (keep it beside the built binary).

## Build & run

```
./build-misdos.sh              # or: win32 | os2 | darwin
bin/misdos                     # uses modem.ini if present, else Local Mode
```

Needs the `mystic_modem` and `mystic_mailer` sources (on the unit path via
the build script) plus the `mdl/` units those modules use (`m_FileIO`,
`m_IniReader`).  Pure FPC RTL otherwise (Crt for the console).

## Note on the modem library

`mystic_modem` runtime-loads its serial/FOSSIL support; with no modem
present the example falls back to **Local Mode** (SPACE to log in locally),
so it runs and demonstrates the full WFC even on a machine with no serial
hardware.
